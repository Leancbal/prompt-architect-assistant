### ¿Qué es este proyecto?

**_Prompt Architect Assistant es una aplicación desarrollada con 
Python & Streamlit cuyo objetivo es ayudar a las personas a crear 
prompts más efectivos para interactuar con modelos de Inteligencia 
Artificial._**

**_Muchas veces los usuarios conocen qué quieren pedirle a una IA, 
pero no saben cómo estructurar correctamente sus instrucciones 
para obtener respuestas claras, precisas y útiles. Este proyecto 
busca resolver ese problema mediante una experiencia guiada que 
transforma una idea inicial en un prompt completo y bien organizado._**

### Objetivo

**_El objetivo principal es aplicar los conocimientos adquiridos sobre 
Fundamentos de Inteligencia Artificial, Prompt Engineering y Programación 
en Python para construir una herramienta práctica que ayude a los usuarios a:_**

* **_Comprender los componentes de un buen prompt._**
* **_Aprender conceptos básicos de Prompt Engineering._**
* **_Generar prompts estructurados para diferentes necesidades._**
* **_Explorar técnicas avanzadas de prompting de forma sencilla._**

### ¿Cómo funciona?

La aplicación se divide en tres secciones:

1. **_📖Bienvenidos_**

    Presentación del proyecto, objetivos, función y alcance.

2. **_📚Información_**

    Espacio educativo donde se presentan conceptos fundamentales relacionados 
    con prompts, componentes de un prompt efectivo y técnicas de Prompt Engineering.

3. **_🛠️Prompt Architect Assistant_**

    Asistente interactivo que guía al usuario mediante preguntas para construir 
    un prompt personalizado. Una vez completado el proceso, el sistema genera un 
    prompt listo para utilizar en modelos de lenguaje como ChatGPT, Gemini, Claude 
    o similares.

### Problema que resuelve

**Muchas personas usan IA así:**

`Hazme una presentación sobre ventas.`

Y reciben resultados mediocres.

**Esta aplicación te guía para convertir esa idea en:**

```
Actúa como consultor comercial senior.

Objetivo:
Diseñar una presentación de ventas.

Contexto:
La presentación será utilizada por una empresa B2B de software.

Pasos:
1. Analizar la audiencia.
2. Identificar puntos de dolor.
3. Proponer soluciones.
4. Construir narrativa persuasiva.

Reglas:
- No inventar datos.
- Utilizar lenguaje profesional.

Formato de salida:
Presentación en diapositivas.

Piensa paso a paso antes de responder.
```

**Mucho mejor.**
